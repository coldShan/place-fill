Element UI 输入框霓虹边框复现页

这个页面专门验证扩展的输入框霓虹边框在 Element UI 场景下是否会出现“视觉层盖在输入框上面，但由于 pointer-events: none 仍可点击输入”的问题。

打开扩展后，先聚焦左侧对照组，再聚焦右侧复现组。右侧容器可动态开启常见的 stacking context 触发条件。

推荐先只开 transform，它最容易稳定制造局部 stacking context。

Control

正常 Element UI 输入框

这个输入框不放进额外 stacking context。它应该是观察霓虹边框正常表现的基线。

  • 预期：霓虹边框在输入框外沿，不会盖到输入框视觉层。
  • 如果这里都异常，说明问题不依赖 Element UI stacking context。
Stress

放进 stacking context 的 Element UI 输入框

这个容器会按上方开关动态附加 transform / filter / opacity。如果扩展的霓虹层挂在根层，而 Element UI 输入框被困在局部上下文里，就容易复现“边框压在上面”的现象。

transform filter opacity
  • 如果只在这里出现覆盖感，基本就是 stacking context 问题。
  • 如果去掉全部开关后恢复正常，说明复现成功。
Debug

定位提示

  • DevTools 看输入框祖先链上是否出现 transform、filter、opacity、contain、isolation、will-change。
  • 确认扩展打标的节点是不是 .el-input__inner，而不是外层 .el-input。
  • 把扩展里的 .ctdp-smartfocus 临时去掉 pointer-events: none，如果输入框立刻点不动，就说明 glow 层真实在上面。

CDN 方式基于 Element UI 官方文档的最小浏览器引入：Vue 2 + Element UI CSS/JS。
